Some simple exercises in propositional logic

The first chapter of Dirk van Dalen's book Logic and Structure has propositional logic as subject. We can solve some of the exercises of this chapter quite easily with MPA.

In exercise 1 on page 20 we are invited to check whether some propositions are valid. Let's check this!

The distribution of MPA contains a couple of example projects. We will open the project vanDalen:

Project > Open Project. In the file open dialog choose folder Examples and file vanDalen.mpf

Click on the project node named vanDalen in the left pane and open the editor with one of the propositions e.g. p20-1h. You should see something like this:



Now we can check whether the proposition is a tautology by using the tasks
Tasks > Truth Table or
Tasks > check VAL

There are more tasks, see Tasks. Play with MPA!

Map Coloring

The problem to check whether a planar map can be colored by three or four colors such that adjacent countries doesn't have the same color, can easily be formulated in the propositional logic:

Let c be a country and r, g, b three colors, we use three propositional atoms c_r, c_g, c_b to express that the country is red, green oder blue. Clearly: a country may just have one of these colors. This is expressed by the proposition
(!c_r|!c_g) & (!c_r|!c_b) & (!c_g|!c_b) & (c_r|c_g|c_b).
In a similar way, we can express that adjacent countries must have different colors.

The project Coloring.mpf contains some map coloring problems. Opening the project and the open the editor with Luxembourg. You should see this:


Obviously it would be exhausting to repeat the same conditions for the different countries. On this account MPA has a built-in macro processor MMP that is compatible with the classical and well-known M4 macro processor.

The first line of the coloring problem Luxembourg is as follows:
dnl needs mmp
This line is a directive that informs MPA to first perform the macro expansion and then the task on the resulting proposition. More about macro processing in MPA, see Macros in MPA.

The MPA distribution contains some pre-defined macros in the macro libraries util.mml and logic.mml. In order to locate these files, MPA has to be informed on the folder of these files, i.e. the folder Examples. To do this, use Options > Include Path and add the folder Example on your machine. You can check the result of macro preprocessing wiht the task .

With the appropriate include path, we can check whether a map cutout of the surroundings of Luxembourg can be colored by 3 colors. Not difficult to guess the answer anyway.

There are more examples, e.g. Australia with 3 colors, U.S with 3 and 4 colors.

Sudoku

The last example found in the folder Examples is Sudoku, which can be encoded in the propositional logic by using 9 atoms for each cell of the 9x9-lattice of sudoku: c122 is true if and only if the cell at line 1 and column 2 has the value "2". We need 9^3 = 729 atoms for a sudoku.

Again, the built-in macro processor does a good job. We put the rules of sudoku into the include file sudoku.m4 and the given values of the puzzle in our proposition.

Given the rules and the values we can check the satisfiability and obtain a solution (if there is one). To see all atoms obstructs the view on the result. With the Options > Settings "Models are shown... ...true variables in tabular form" we obtain a more consie result, like this:
