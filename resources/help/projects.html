Projects and Files

The propositions in LWF are organized via projects. One project is one file on disc; that means one file contains one or more propositions.

The menu "Project" provides the actions to work with projects and files.

Projects

Proposition are organized in Projects. Each project is stored in a file named such as the project and extension ".mpf" (MPA project file). The propositions in a project have a unique name.

A project file is organized as follows:

  • A header (as a C-style comment), which can be edited by double clicking the project name.
  • A sequence of propositions. The beginning of a task is marked in the project file by the line
    phi::<name of proposition>.
    Proposition can be edited by double clicking the name of the proposition.

M4 Files

NOTICE: Right now, M4 files can be used in propositions, but can not be edited. Please use a normal text editor for that instead.

MPA uses two kinds of M4 files:

  • M4 files for inclusion in proposition. M4 files are used to provide reusable macros for use in the propositions. M4 files have the extension ".m4".
  • The MNI Macro Processor MM not only implements the functionality of M4 but also has a mechanism to extend the built-in macros by macros written in the Java programming language. Such extension are included in the form of Macro Library files, which have the extension ".mml". See Macros in MPA.
